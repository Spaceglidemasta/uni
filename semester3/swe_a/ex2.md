# Exercise 2

### 1
- Customers often face difficulties in articulating their needs, and the problem
may not be fully understood, leading to potential misunderstandings or incorrect
requirements. 
- The requirements analysis phase is rarely influenced by domain-specific
vocabulary, as it typically aligns with the development team's technical jargon.

### 2
- _"The program should be fast"_: How fast should it be? How is this supposed to be achieved?
- _"The program should never crash"_: __Never__ is not possible, how safe is the program supposed to be? What function can / need to be cut because of this?
- _"The product needs to be attractive for new users"_: How is this supposed to be achieved? How _new_ is a new user?
- _"The System should have a Dark mode. [...] The system should always have a white background"_: These requirements are conflicting with each other.
