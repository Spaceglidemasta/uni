### 2
The analysis phase is very important in most big projects, because it determents how the code-logic is structured, which is very important for most programs that are even slightly dependent on efficiency.

Skipping the Analysis phase can make program slow and very hard to debug and extend. Despite my projects beeing very small, I still needed to restructure big parts of my code, just because I didn't think about extending the program to more functionalities when i started coding it.

### 3
Starting with a Design model can be a very bad idea, because sometimes the Design is dependent on the code-logic. Basing the code on the Design can fail the whole project, as efficient code is way more important to a program then small design differences.

Design can easily be changed in the future, the whole logic of the program cannnot.
